# SBPE: Set-Based Parameter Explorer

![C++](https://img.shields.io/badge/C%2B%2B-17-00599C?logo=cplusplus&logoColor=white)
![Build](https://img.shields.io/badge/dependencies-none%20(STL%20only)-informational)
![Status](https://img.shields.io/badge/status-prototype-orange)
![License](https://img.shields.io/badge/license-MIT-yellow)

SBPE（Set-Based Parameter Explorer）は、**セットベース設計**の考え方をもとに、2次元パラメータ空間を探索・可視化するC++製プロトタイプです。

「唯一の最適値」を1点だけ求めるのではなく、**どの範囲のパラメータなら結果が許容できるのか**をスコアマップとして可視化することを目的としています。

このリポジトリは、下記の [白線検知プロジェクト（LaneDetection-CPP）](#related-project-lanedetection-cpp) でのパラメータ調整の煩雑さを解決するために着想・実装したツールです。

## Overview

このプロジェクトでは、ある評価関数に対して2つのパラメータを変化させながら探索を行い、各点の評価値をスコアとして可視化します。

主な目的は、唯一の最適値だけを探すのではなく、

* どの範囲のパラメータが許容されるのか
* target value に近い領域がどこに分布しているのか
* パラメータを変更したとき、評価結果がどのように変化するのか
* 手作業でのパラメータ調整をどのように体系化できるか

を観察することです。

## Background

このプロジェクトは、白線検知システム [LaneDetection-CPP](https://github.com/mailryogot85-ops/LaneDetection-CPP) を制作していた際の問題意識から始まりました。

白線検知では、Cannyエッジ検出やHough変換のパラメータによって検出結果が大きく変化します。
しかし、実際にパラメータを調整する際には、一つずつ値を変更して結果を確認する必要があり、非常に煩雑でした。

特に、以下のような課題を感じました。

* パラメータ同士が相互作用する（片方を変えるともう片方の最適値も変わる）
* 少し値を変えるだけで検出結果が大きく変化し、変化の傾向を直感で予測しづらい
* 手作業のtry & errorでは、探索した範囲・していない範囲を体系的に把握できない
* 「たまたま上手くいった一点」ではなく、「どの範囲まで崩れずに機能するか」を知りたい

これは単なる白線検知固有の問題ではなく、複数パラメータを持つシステムを調整する際に一般的に起こりうる課題だと考えました。
そこで、大学の講義で学んだセットベース設計の考え方から影響を受け、単一の最適点ではなく、成立可能なパラメータ領域全体を探索・可視化する**独立した汎用ツール**としてSBPEを構想・実装しました。

## Concept

SBPEでは、パラメータ空間上の各点を評価し、その結果をスコアとして扱います。

現在の実装では、target value に近いほど高いスコアになるよう、Gaussianベースのスコアリングを用いています。

スコアは `0.00` から `1.00` の範囲で表されます。

* target value に近い点ほど高スコア
* target value から離れるほどスコアが低下
* threshold 以上の点を feasible region として扱う

これにより、単に最も良い一点を探すのではなく、許容できるパラメータ領域を観察することを目指しています。

---

## Current Implementation

現在のプロトタイプでは、以下の機能を実装しています。

* 2変数二次関数を対象としたパラメータ探索
* Gaussianベースのスコアリング
* threshold を満たす点の判定
* terminal上での score map 表示
* target value 近傍の探索
* feasible region の面積・割合の算出

現時点では、実画像とはまだ直接統合していません。
まずは、評価関数・スコアリング・許容領域の可視化という基本構造を確認するための実験的な実装です。
