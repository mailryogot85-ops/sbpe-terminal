# SBPE: Set-Based Parameter Explorer

SBPE（Set-Based Parameter Explorer）は、セットベース設計の考え方をもとに、2次元パラメータ空間を探索・可視化するためのC++製プロトタイプです。

現在はterminal上で動作する初期実装であり、2変数関数に対してスコアリングを行い、許容範囲を満たす領域を可視化・集計することを目的としています。

---

## Overview

このプロジェクトでは、ある評価関数に対して2つのパラメータを変化させながら探索を行い、各点の評価値をスコアとして可視化します。

主な目的は、唯一の最適値だけを探すのではなく、

* どの範囲のパラメータが許容されるのか
* target value に近い領域がどこに分布しているのか
* パラメータを変更したとき、評価結果がどのように変化するのか
* 手作業でのパラメータ調整をどのように体系化できるか

を観察することです。

---

## Background

このプロジェクトは、OpenCVを用いた白線検知システム `LaneDetection-CPP` を制作していた際の問題意識から始まりました。

白線検知では、Cannyエッジ検出やHough変換のパラメータによって検出結果が大きく変化します。
しかし、実際にパラメータを調整する際には、一つずつ値を変更して結果を確認する必要があり、非常に煩雑でした。

特に、以下のような課題を感じました。

* パラメータ同士が相互作用する
* 少し値を変えるだけで結果が大きく変化する
* 手作業では探索範囲を体系的に把握しづらい
* 「最適値」だけでなく「どの範囲まで許容できるか」を見たい

そこで、大学の講義で学んだセットベース設計の考え方から影響を受け、単一の最適点ではなく、成立可能なパラメータ領域全体を探索・可視化するツールとしてSBPEを構想しました。

---

## Concept

SBPEでは、パラメータ空間上の各点を評価し、その結果をスコアとして扱います。

現在の実装では、target value に近いほど高いスコアになるよう、Gaussianベースのスコアリングを用いています。

スコアは `0.00` から `1.00` の範囲で表されます。

* target value に近い点ほど高スコア
* target value から離れるほどスコアが低下
* threshold 以上の点を feasible region として扱う

これにより、単に最も良い一点を探すのではなく、許容できるパラメータ領域を観察することを目指しています。

---

## Current Implementation

現在のプロトタイプでは、以下の機能を実装しています。

* 2変数二次関数を対象としたパラメータ探索
* Gaussianベースのスコアリング
* threshold を満たす点の判定
* terminal上での score map 表示
* target value 近傍の探索
* feasible region の面積・割合の算出

現時点では、実画像やOpenCVとはまだ直接統合していません。
まずは、評価関数・スコアリング・許容領域の可視化という基本構造を確認するための実験的な実装です。

---

## Example Output

SBPEは、各パラメータ点のスコアをterminal上にマップとして出力します。

```text
Score Map
0.12 0.28 0.45 0.28 0.12
0.28 0.61 0.82 0.61 0.28
0.45 0.82 1.00 0.82 0.45
0.28 0.61 0.82 0.61 0.28
0.12 0.28 0.45 0.28 0.12
```

このように、target value に近い領域ほど高いスコアとして表示されます。

---

## Processing Flow

現在の処理の流れは以下の通りです。

```text
Parameter Range
    ↓
Evaluate each point
    ↓
Calculate raw value
    ↓
Convert raw value to score
    ↓
Check threshold
    ↓
Visualize score map
    ↓
Calculate feasible region
```

## Related Project

### LaneDetection-CPP

SBPEは、OpenCVを用いた白線検知システム `LaneDetection-CPP` の開発中に感じた課題から派生したプロジェクトです。

LaneDetection-CPPでは、Cannyエッジ検出やHough変換のパラメータ調整を手作業で行う必要がありました。
SBPEは、そのようなパラメータ調整をより体系的に扱うための探索・可視化ツールとして構想しています。

## Limitations

現在の実装には、以下の制約があります。

* terminalベースの初期プロトタイプである
* 評価対象は2変数関数に限定されている
* 実画像を用いた評価には未対応
* OpenCVとはまだ統合していない
* GUIや連続的なヒートマップ表示には未対応
* 実用的な自動パラメータ最適化ツールではなく、探索・可視化の実験段階である

---

## Goal

SBPEの目的は、パラメータ調整を単なる試行錯誤で終わらせず、探索空間として可視化・分析できる形にすることです。

画像処理やロボティクスでは、パラメータのわずかな違いが結果に大きく影響することがあります。
そのため、唯一の最適値だけでなく、安定して成立するパラメータ領域を把握することが重要だと考えています。

将来的には、OpenCVや実画像処理パイプラインと連携し、人間が感覚的に行っているパラメータ調整を、より体系的に探索・評価できるツールへ発展させることを目指しています。
